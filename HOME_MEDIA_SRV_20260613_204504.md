# Reporte Ejecutivo de Auditoría de Seguridad

## 1. Resumen Ejecutivo
**Score de Seguridad General:** 30 / 100

*Justificación del Score:* La puntuación de 30/100 refleja una postura de seguridad deficiente, principalmente debido a la exposición de servicios de alto riesgo como Samba (SMB/CIFS) y una gran cantidad de puertos asociados a Docker, lo que amplía significativamente la superficie de ataque. Se identificaron configuraciones de usuario inseguras, como una cuenta de sistema con un shell interactivo inapropiado y el acceso directo de 'root'. Además, la presencia de un servicio 'systemd' escuchando en un puerto no estándar y la exposición de herramientas de monitoreo contribuyen a la baja calificación.

### Evaluación Gerencial
El servidor auditado presenta una postura de seguridad débil que requiere atención inmediata. Se han detectado múltiples vulnerabilidades críticas y de alto impacto, incluyendo la exposición de servicios de compartición de archivos (SMB/CIFS) que son blancos frecuentes de ataques, y una extensa superficie de ataque generada por numerosos puertos abiertos de Docker. La gestión de usuarios muestra deficiencias, con una cuenta de sistema configurada de forma insegura y la posibilidad de acceso directo como 'root'. Estas debilidades representan un riesgo significativo para la integridad y confidencialidad de los datos, así como para la disponibilidad del servicio. Se recomienda priorizar las mitigaciones propuestas para fortalecer la seguridad del sistema.

## 2. Hallazgos Prioritarios
- Exposición de servicios SMB/CIFS (puertos 139, 445 TCP y 137, 138 UDP) a la red.
- Gran superficie de ataque debido a la exposición de múltiples puertos de docker-proxy.
- Cuenta de sistema User_1 (UID 4) con shell interactiva /bin/sync.
- Servicio systemd escuchando en el puerto 9090/tcp.
- Acceso directo de root con shell /bin/bash.

## 4. Recomendaciones Técnicas de Mitigación

### 🛠️ Restringir o deshabilitar Samba
*Deshabilitar los servicios Samba si no son estrictamente necesarios. Si son necesarios, restringir el acceso únicamente a IPs de confianza mediante un firewall.*

```bash
sudo systemctl stop smbd nmbd
sudo systemctl disable smbd nmbd
sudo ufw deny from any to any port 137,138,139,445 proto tcp
sudo ufw deny from any to any port 137,138 proto udp
sudo ufw reload
```

### 🛠️ Revisar y minimizar puertos expuestos por Docker
*Auditar todos los contenedores Docker para identificar qué servicios están exponiendo puertos. Minimizar la exposición de puertos a solo aquellos estrictamente necesarios y restringir el acceso mediante un firewall o configuraciones de red de Docker.*

```bash
sudo docker ps -a
sudo docker inspect <container_id>
sudo nano /etc/docker/daemon.json
sudo systemctl restart docker
```

### 🛠️ Cambiar shell de User_1 a nologin
*Modificar el shell de la cuenta User_1 a /usr/sbin/nologin para prevenir cualquier intento de login interactivo, ya que es una cuenta de sistema y no debería tener acceso de shell.*

```bash
sudo usermod -s /usr/sbin/nologin User_1
```

### 🛠️ Identificar y asegurar el servicio en 9090/tcp
*Investigar qué proceso real está escuchando en el puerto 9090/tcp y por qué se reporta como systemd. Si es un servicio legítimo, asegurar su configuración y restringir el acceso. Si es innecesario o malicioso, deshabilitarlo o eliminarlo.*

```bash
sudo lsof -i :9090
sudo netstat -tulnp | grep 9090
sudo systemctl status <servicio_identificado>
sudo ufw deny 9090/tcp
```

### 🛠️ Deshabilitar login directo de 'root' vía SSH
*Configurar el servidor SSH para deshabilitar el login directo de root. En su lugar, los administradores deben iniciar sesión con una cuenta de usuario regular y luego usar sudo para tareas administrativas. Asegúrese de tener un usuario con permisos sudo configurado antes de aplicar este cambio.*

```bash
sudo nano /etc/ssh/sshd_config
sudo systemctl restart sshd
```

### 🛠️ Actualizar el kernel a la última versión estable
*Mantener el kernel del sistema operativo actualizado es crucial para aplicar parches de seguridad que corrigen vulnerabilidades conocidas (CVEs) y mejorar la estabilidad general del sistema.*

```bash
sudo apt update
sudo apt upgrade
sudo apt dist-upgrade
sudo reboot
```

### 🛠️ Restringir acceso a puertos de PCP
*Limitar el acceso a los puertos utilizados por Performance Co-Pilot (PCP) a solo las direcciones IP de los sistemas de monitoreo autorizados para evitar la exposición de información del sistema.*

```bash
sudo ufw deny from any to any port 4330,44321,44322,44323 proto tcp
sudo ufw reload
```

### 🛠️ Asegurar o deshabilitar el servicio Postfix
*Si el servidor no está destinado a ser un servidor de correo, deshabilitar Postfix y bloquear el puerto 25. Si es un servidor de correo, asegurar su configuración para prevenir el uso como relay abierto y aplicar las mejores prácticas de seguridad para SMTP.*

```bash
sudo systemctl stop postfix
sudo systemctl disable postfix
sudo ufw deny 25/tcp
sudo ufw reload
```
